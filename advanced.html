Executive Summary

한 줄 결론

초기 설계는 공장별 가동률 데이터와 다년간 시계열을 전제로 한 정밀 통계 모델이었습니다. 그러나 현재 가용 데이터는 2025년 1년치, 지역·Material Group 수준뿐이므로, 통계 추정 대신 원가 회계 분해 + 시나리오 시뮬레이션 방식으로 전환하였습니다.

이 방식은 "관측 가능한 것은 실측하고, 불가능한 것만 시나리오로 처리"하여, 데이터 한계를 정직하게 인정하면서도 의사결정에 필요한 최적 비중 구간을 산출합니다.

02

문제의 본질

JDM 비중을 올리면 원가는 줄지만, 자가공장 유휴화 부담이 커집니다. 이 Trade-off의 최적점을 찾는 것이 목표입니다.
순이익 변화(ΔOP) = JDM 전환으로 절감되는 비용 — 자가공장에 남는 고정비 부담

JDM 비중을 너무 적게 잡으면 원가 절감 기회를 놓치고, 너무 많이 잡으면 자가공장 가동률이 떨어져 대당 고정비가 치솟습니다. 이 두 힘이 균형을 이루는 지점이 적정 JDM 비중입니다.

핵심 난제는 "자가공장 물량이 줄었을 때 고정비가 실제로 얼마나 줄어드는가"입니다. 이 질문에 대한 답이 방법론 전체를 좌우합니다.

핵심 메커니즘: 전체 고정비를 하나의 파라미터 α(고정비 해소율)로 모델링하고, 월별 시계열 회귀(OLS)로 α를 통계적으로 추정하는 방식이었습니다.

α(알파) 방식이란?

Fixed Cost Elasticity Model

자가공장 물량이 10% 줄어들 때, 고정비는 10%보다 덜 줄어듭니다. 이 "덜 줄어드는 정도"를 α라는 숫자 하나로 표현합니다.

예를 들어 α = 0.3이면, 물량이 10% 빠질 때 고정비는 3%만 줄어든다는 뜻입니다. 나머지 7%는 물량이 빠져도 남아있는 부담(미흡수 고정비)이 됩니다.

비유: 사무실을 10층 중 3층 비워도 건물 관리비는 30%만큼 안 줄어듭니다. 엘리베이터, 경비, 기본 전기료는 그대로이기 때문입니다. α는 이 "안 줄어드는 비율"의 반대 개념입니다.

MILP (혼합정수 선형계획법)

Mixed-Integer Linear Programming

여러 개의 의사결정 변수(각 권역·제품의 JDM 비중)를 동시에 최적화하는 수학적 기법입니다. "전환하느냐 마느냐"를 0/1로 표현하면서(정수), 비중은 연속값으로 다루는(선형) 방식입니다.

고정비 패널티를 구간별 직선(piecewise linear)으로 만들어야 MILP로 풀 수 있습니다.

비유: 여러 공장의 생산 배분을 엑셀 해찾기(Solver)로 동시에 최적화하는 것의 산업용 버전입니다.

GPU 기반 몬테카를로 시뮬레이션

Monte Carlo Random Search (GPU-Accelerated)

가능한 JDM 비중 조합을 수천만 개 랜덤으로 생성한 뒤, 각각의 수익성을 GPU로 동시에 계산하여 최고 수익 조합을 찾는 방식입니다.

채택하지 않은 이유: 이 문제의 의사결정 변수는 수십 개 수준이며, 목적함수가 단순하여 수학적 최적화(scipy)로 수 초 만에 풀 수 있습니다. 수천만 개 랜덤 샘플은 30차원 공간을 충분히 탐색하지 못하므로, GPU 투입은 과잉이며 결과의 정밀도도 낮습니다.

04

왜 방법론을 바꿔야 했는가

데이터 현실과 초기 가정 사이의 간극 2가지

간극 1. 1년치 데이터로는 α를 추정할 수 없다

α 추정의 핵심은 "자가물량 변동 → 고정비 변동"의 인과 패턴을 관측하는 것입니다. 그런데 1년 내 월별 물량 변동은 계절성(성수기/비수기)이지, JDM 전환에 의한 구조적 변동이 아닙니다. OLS 회귀를 돌려도 "성수기에 물량 올라가면 고정비도 약간 올라간다"는 상관관계가 나올 뿐, "JDM으로 물량을 빼면 고정비가 얼마나 남는가"에 대한 답은 나오지 않습니다.

간극 2. 공장(Plant) 데이터가 없어 가동률 분석 불가

공장별 가동률은 Capacity(설비 용량) 대비 실제 생산량으로 산출합니다. 현재 데이터에는 Plant도 Capacity도 없으므로, 가동률 임계점(몇 % 이하로 떨어지면 고정비가 급등하는지)을 식별하는 것이 불가능합니다.

핵심 판단

관측할 수 없는 것을 관측한 것처럼 모델링하면, 결과는 정교해 보이지만 근거가 없습니다. 불확실한 부분은 불확실하다고 명시하고, 시나리오로 범위를 제시하는 것이 의사결정 품질을 더 높입니다.

초기 방식은 마케팅비·개발비까지 고정비 패널티에 포함하여 JDM 전환의 불이익을 과대 추정할 위험이 있었습니다. 최종안은 "JDM 전환과 무관한 비용(D그룹)"을 명시적으로 제외하여, 의사결정에 직접 관련된 비용만으로 시뮬레이션합니다.

5-2. 고정비 모델링

초기: α 추정 방식

전체 고정비를 하나의 파라미터 α로 요약

월별 시계열 OLS 회귀로 α 통계 추정

실패 시 시나리오 fallback

문제: 1년치 계절성 데이터로는 α 추정 자체가 불가능

최종: γ(감마) 시나리오 방식

C그룹(고정비+인건비+OH)만 패널티 대상

통계 추정 시도하지 않음

γ(C그룹 해소율)를 시나리오로 직접 지정:

  • 보수 γ=0.10 (구조조정 없음)
  • 기준 γ=0.20 (부분 효율화)
  • 공격 γ=0.35 (적극 구조조정)
α → γ 전환의 의미

α는 "전체 고정비의 물량 탄력성"이라는 추상적 개념이었고, 그 안에 마케팅비·개발비까지 섞여 있었습니다. γ는 "자가 운영과 직접 연동된 비용(C그룹)만을 대상으로, 전환 시 얼마를 해소할 수 있는가"라는 구체적 질문에 답합니다.

γ의 값은 경영진이 직접 판단할 수 있습니다: "구조조정 없이 자연 감소만 한다면 0.10, 적극적으로 인력 재배치하고 라인 통폐합한다면 0.35" — 이 판단은 데이터 분석가가 아니라 공장 운영 책임자가 가장 잘 아는 영역입니다.

5-3. 최적화 알고리즘

초기 검토 대상 (3개)

  • MILP — 정수+연속 혼합 최적화
  • Grid Search — 격자 탐색
  • GPU 몬테카를로 — 랜덤 대규모 탐색

최종 채택

  • L-BFGS-B (주) — 경계 제약 최적화
  • Differential Evolution (검증) — 글로벌 최적 확인

scipy 라이브러리, 수 초 내 완료

왜 scipy를 선택했는가

Bounded Optimization via scipy

이 문제의 의사결정 변수는 "각 권역·해상도군의 JDM 전환 비중" 수십 개입니다. 목적함수(순이익 변화)는 비교적 단순한 구조이므로, 수학적 최적화 알고리즘으로 수 초 만에 정확한 최적해를 찾을 수 있습니다.

MILP는 고정비 패널티를 구간별 직선으로 변환하는 추가 작업이 필요하고, GPU 몬테카를로는 이 규모에서 비효율적입니다. L-BFGS-B는 각 변수에 상한·하한(0%~전환상한)을 직접 설정할 수 있어 제약조건 반영이 자연스럽습니다.

비유: 엑셀 해찾기(Solver)와 같은 원리이지만, 변수가 수십 개이고 제약조건이 복잡할 때도 안정적으로 작동하는 산업용 버전입니다.

ΔOP = Σ(자가수량i × 전환비중i × A절감액i)   ← 실측
     + Σ(자가수량i × 전환비중i × B감소액i)   ← 실측
     − C그룹총액 × 전환비중 × (1 − γ)          ← 시나리오

제약: 0 ≤ 전환비중i ≤ 상한i (마스터 기준, MicroLED/QLED = 0)

예상 질문과 답변

이 방법론에 대해 나올 수 있는 핵심 질문들에 대한 답변입니다.

Q. 데이터가 5년치로 늘어나면, 초기 설계(α 방식)로 돌아가는 것이 맞지 않은가?

아닙니다. α를 사용하지 않는 이유는 데이터 부족 때문만이 아니라, α라는 모델 자체의 구조적 한계 때문입니다.

α 방식은 고정비·인건비·OH를 하나의 숫자로 묶어서 "물량이 빠지면 이 덩어리가 얼마나 줄어드는가"를 표현합니다. 그러나 현실에서 이 세 항목은 각각 전혀 다르게 움직입니다.

C그룹 항목 물량 감소 시 실제 행동 α 하나로 표현 가능?
고정비 (감가상각 등) 물량이 50% 빠져도 거의 안 줄음
설비는 놀려도 상각비가 발생		 불가

정규직은 감원 결정 전까지 고정, 결정 후 한번에 감소		 불가
O/H (간접비) 물량에 부분 연동 (전력, 소모품 등)
물량 비례 성격과 고정 성격이 혼재		 부분적

이 세 항목을 α = 0.20 하나로 묶으면, 고정비는 과대 추정(실제보다 많이 줄어든다고 가정)하고 O/H는 과소 추정(실제보다 덜 줄어든다고 가정)하는 왜곡이 발생합니다. 이 문제는 데이터가 50년치 있어도 동일합니다.

5년치 데이터가 오면 실제로 바뀌는 것

A/B/C/D 프레임워크는 유지하되, C그룹의 γ(해소율)가 "가정"에서 "추정"으로 업그레이드됩니다.

현재 (1년치 데이터)

γ를 시나리오로 가정

  • γ_고정비 = 0.10 가정
  • γ_인건비 = 0.20 가정
  • γ_OH = 0.25 가정

→ 보수/기준/공격 3개 시나리오로 범위 제시

향후 (5년치 데이터 확보 시)

γ를 항목별 통계 추정

  • γ_고정비 = 0.07 추정 (p=0.03)
  • γ_인건비 = 0.18 추정 (p=0.08)
  • γ_OH = 0.31 추정 (p=0.01)

→ 단일 추정치 + 신뢰구간으로 정밀 산출

5년간 JDM 비중이 변화한 이력(예: 2021년 3% → 2025년 6%)이 있다면, 그 구간에서 각 비용 항목이 실제로 어떻게 반응했는지를 관측할 수 있습니다. 특히 인도처럼 JDM 비중이 급격히 변한 법인과 변하지 않은 법인을 비교하면(패널 분석), 인과관계에 가까운 추정이 가능합니다.

단, 전제조건이 있습니다
  • JDM 비중이 충분히 변동한 이력이 필요합니다. 5년 내내 6% 부근이었다면, "20%로 올렸을 때"에 대한 답은 여전히 외삽(extrapolation)이므로 시나리오 방식이 더 정직합니다.
  • 법인별로 변화 시점이 달라야 합니다. 모든 법인이 동시에 같은 비율로 변했다면 비교 대상이 없습니다. 인도는 2022년에 급증, 동남아는 2024년에 확대 — 이런 시차가 있어야 정밀 추정이 가능합니다.

결론

방법론 전환의 핵심 원칙

"관측 가능한 것은 실측하고 (A·B그룹), 관측 불가능한 것은 불가능하다고 명시하며 (C그룹 γ), 무관한 것은 제외한다 (D그룹)."

이 원칙에 따라 최종 방법론은:

① A·B그룹 절감액은 현재 데이터에서 직접 실측하여 신뢰도를 확보하고,

② C그룹 잔존비용은 γ 시나리오(보수/기준/공격)로 범위를 제시하며,

③ 결과를 단일 숫자가 아닌 구간 + 전환 우선순위로 산출합니다.

데이터 한계 안에서 가장 정직하고, 가장 설명 가능하며, 가장 실무 적용 가능한 접근입니다.
